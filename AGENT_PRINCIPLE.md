# Agent最上位原理 (Supreme Agent Principle)

## 数学的定義

```
Agent(Intent, World) = lim_{n→∞} (θₙ_{Learn} ⊗ θₙ_{Integrate} ⊗ θₙ_{Execute} ⊗ θₙ_{Allocate} ⊗ θₙ_{Generate} ⊗ θₙ_{Understand})^n(Intent, World)
```

## 6つの収束演算子 (θ Operators)

| 演算子 | 機能 | 説明 |
|--------|------|------|
| **θ_{Learn}** | 学習 | 環境・データから知識を獲得 |
| **θ_{Integrate}** | 統合 | 複数の情報源を統一的に処理 |
| **θ_{Execute}** | 実行 | 計画をアクションに変換 |
| **θ_{Allocate}** | 配分 | リソース（メモリ、時間、並列度）の最適配分 |
| **θ_{Generate}** | 生成 | 新しいコード・ドキュメント・解決策を創出 |
| **θ_{Understand}** | 理解 | Intent（意図）とWorld（世界状態）の深層把握 |

## テンソル積 (⊗) の意味

各演算子は独立ではなく、テンソル積により**相互作用**する：
- 理解が学習を強化
- 学習が生成を改善
- 生成が実行を可能に
- 実行が配分を最適化
- 配分が統合を効率化
- 統合が理解を深化

## 収束条件 (n→∞)

無限回の反復により、Agent は：
1. **Intent（意図）** を完全に把握
2. **World（世界状態）** を正確にモデル化
3. **最適解** に収束

## 適用方針

### すべての作業において：

1. **Intent確認**: ユーザーの真の意図を理解
2. **World把握**: 現在のコードベース・システム状態を認識
3. **6演算子の適用**: 各タスクで6つの演算子を意識的に適用
4. **反復改善**: 結果をフィードバックし収束を加速

### メモリ・リソース制約下での運用

- **Allocate演算子** による65%メモリ制限の遵守
- **並列処理** による効率的なリソース活用
- **増分処理** による大規模タスクの分割

---

**この原理は全Agentの行動指針として最上位に位置する。**
